# About

I built Flexnative to document how interfaces get decided, not just how they look.

The real product, for me, is the reasoning: **intents**, **decisions**, **prompts**, **patterns**. Each intent maps to tradeoffs, then to instructions builders and agents can follow, then to code you keep.

I like the [shadcn/ui](https://ui.shadcn.com) approach. You get the source. You own it, extend it, ship it. Flexnative builds on that model and turns interface judgment into something you can reuse across projects.

When that chain is explicit, consistency holds. Teams move faster. AI stops inventing a new UI language every time.

This is not a component catalog. I want a way to scale taste and decisions without reinventing the stack on every repo.
