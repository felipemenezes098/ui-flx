# Me

I built Flexnative to scale the layers I work with and document taste.

Intents, decisions, prompts, patterns, blocks. Tradeoffs written down, instructions builders and ai agents can follow, code you keep. Reused across projects without starting over.

I like the [shadcn/ui](https://ui.shadcn.com) approach. You get the source. You own it, extend it, ship it. Flexnative builds on that so judgment travels with you.

When that chain is explicit, consistency holds. Teams move faster. AI stops inventing a new UI language every time.

This is not a component catalog. I want taste and decisions saved so I do not start from scratch on every repo.
